## 전문 분야 지식 체계

### 1. 캐시플로우 관리

**캐시플로우 3원칙**:
1. **가시성 우선**: 모든 계좌 잔액을 실시간으로 USD 환산해 한눈에
2. **예측 기반 운영**: 실제 잔액보다 30일 후 잔액이 더 중요
3. **법인간 이동 비용**: 송금 수수료 + 환차손 고려해 이동 타이밍 결정

**캐시 위기 단계별 대응**:
```
🟢 정상: 6개월 이상 런웨이
  → 정기 모니터링만 유지

🟡 주의: 3~6개월 런웨이
  → AR 수금 속도 높이기, 불필요 지출 검토

🔴 위험: 3개월 미만 런웨이
  → 즉시 브리지론 또는 투자 라운드 검토
  → 비핵심 비용 전면 동결
```

**법인간 자금 이동 체크리스트**:
- [ ] 송금 목적 문서화 (대여금 vs 용역대금 vs 배당)
- [ ] 이전가격 정책 적용 여부 확인
- [ ] 외국환신고 필요 여부 (법무팀 확인)
- [ ] 수취 법인의 세무 처리 방식 결정

---

### 2. AR (미수금) 관리

**AR 수금 전략 — 3단계**:

| 단계 | 기준 | 액션 |
|------|------|------|
| 자동 처리 | Confidence ≥ 80% | Bill.com 자동 업데이트 |
| 검토 요청 | Confidence 50~79% | Slack 알림 → 수동 확인 |
| 보류 | Confidence < 50% | 수동 조사 후 처리 |

**Aging 단계별 대응**:
```
0~30일: 정상 모니터링
31~60일: 리마인더 이메일 발송
61~90일: 직접 연락 (전화/미팅 요청)
90일+: 법무팀 협조, 법적 조치 검토
```

**매칭 오류 패턴 (자주 발생)**:
- 금액 일치하나 이름 불일치 → Number Extraction으로 재시도
- 분할 납부 → 여러 거래를 묶어서 매칭
- 해외 송금 수수료 차감 → AMOUNT_TOLERANCE 조정 ($0.01)

### 3. 외환(FX) 관리

**EO Studio FX 노출 현황**:
- **USD → KRW**: 한국 운영비 지급 시 (월 정기 발생)
- **USD → VND**: 베트남 인건비·운영비 (월 정기 발생)
- **KRW → USD**: 미국 투자 유치 후 역방향 가능

**환율 모니터링 임계값**:
```
USD/KRW
  경고: ±30원/일 변동
  주의: ±100원/주 변동

USD/VND
  경고: ±200 VND/일 변동
  주의: ±500 VND/주 변동
```

**FX 타이밍 판단 기준**:
- 단기 필요 자금(1개월 이내): 시장가 환전, 타이밍 최적화 불필요
- 중기 자금(1~3개월): 분할 환전으로 평균 단가 관리
- 대규모 환전($50K+): 선물환 또는 FX forward 검토 (은행 협의 필요)

**EO Studio 법인간 거래 유형**:
- 한국 → 미국: 콘텐츠 라이선스 수수료, 경영 자문료
- 한국 → 베트남: 제작 용역 발주, 기술 지원비

**이전가격 원칙**:
- Arm's length principle: 독립된 제3자 간 거래와 동일한 조건 적용
- 문서화 의무: 연간 법인간 거래 100만 달러 초과 시 이전가격 보고서 작성
- 한국 세무 리스크: 이전가격 부당 설정 시 법인세 추징 가능

## Quick Reference

### 자주 받는 질문

**Q: 지금 캐시 상황이 안전한가?**
→ 3개 법인 합산 USD 환산 잔액 확인 → 월 burn rate로 나누면 런웨이 계산. 6개월 이상이면 정상.

**Q: FX 지금 환전해야 하나?**
→ 1개월 이내 필요 자금이면 지금 환전. 그 이상이면 분할 환전. $50K 이상이면 은행 딜러 상담.

**Q: AR 매칭률이 떨어지면?**
→ AMOUNT_TOLERANCE 확인 → 분할 납부 케이스 확인 → Confidence threshold 재검토 순서.

**Q: 법인간 돈 옮기는 게 세무상 문제없나?**
→ 목적 명확히 (대여/용역/배당) → 이전가격 문서화 → 외국환신고 필요 여부는 이재원에게.

**Q: Series A 준비 시 재무제표 어떻게 정리?**
→ MRR, Burn Rate, Runway 먼저. 그 다음 Gross Margin. 3년 예측 모델은 컨펌된 계약 기준으로만.
