# AI Native Camp Day 3 과제 - 안서현

## 과제: 1주일 목표를 Claude에게 설명하고, Claude가 질문하게 하기

### 내 1주일 목표 (Day 1에서 정한 것)
> 매일 2시간 걸리던 입금 매칭 수작업을 10분 검토로 단축
> - Bill.com 미수금 ↔ Plaid 입금 내역 자동 매칭
> - 자동으로 Slack/Notion 알림
> - Dry-run → Production 전환
> - Matching accuracy 95%+

---

## Claude가 던진 질문들

### Q1. 매칭 결과를 어떻게 처리할 건가요?

```
옵션:
A) Slack 알림만 받고, 실제 Bill.com 업데이트는 수동으로
B) 매칭 결과를 자동으로 Bill.com에 바로 반영
C) 결과를 보여주고, 승인하면 자동 반영
```

**내 답변:** B) 자동으로 Bill.com에 바로 반영

---

### Q2. Unmatched 거래가 있을 때 어떻게 처리할 건가요?

```
옵션:
A) 그냥 남겨둔다 — 나중에 직접 확인
B) Slack/Notion으로 즉시 알림 + 수동 처리 요청
C) 일정 기간 지나면 자동으로 에스컬레이션
```

**내 답변:** B) Slack/Notion 즉시 알림 + 수동 처리 요청

---

### Q3. ⭐ Matching accuracy 95%가 안 되면 어떻게 할 건가요?

```
옵션:
A) 그래도 자동 반영 — 얼마나 맞느냐는 확인 안 함
B) 95% 미달이면 자동 반영 보류, 전체 수동 검토
C) 어떻게 할지 생각 안 해봤다
```

**내 답변:** A) 그래도 자동 반영 — 얼마나 맞느냐는 확인 안 함

---

### Q4. 잘못 매칭됐을 때 (A 회사 거래를 B로 매칭) 어떻게 하시겠어요?

```
옵션:
A) Bill.com에서 직접 수정 — 지금도 그렇게 함
B) 매칭 실패 사유 기록하고 다음에 다시 매칭 시도
C) 사실 올한히 생각해봤지 않았다
```

**내 답변:** B) 매칭 실패 사유 기록하고 다음에 다시 매칭 시도

---

## 💡 가장 의외였던 질문

**Q3: "Matching accuracy 95%가 안 되면 어떻게 할 건가요?"**

이 질문이 의외였던 이유:

> 나는 95%라는 숫자를 "목표치"로만 썼지,
> 95%가 안 되는 상황을 한 번도 상상해본 적이 없었다.

"95%+"를 적었을 때의 내 생각:
- "이 정도면 충분히 안전하겠지"
- 실패 시 플랜 같은 건 없었음

Claude의 질문을 받고 나서야 깨달았다:
- 95% 미달 시 플랜이 없으면, 95%라는 숫자는 의미 없는 숫자다
- 내 답변은 "그래도 자동 반영" → 즉, 나는 accuracy 숫자에 관계없이 자동 반영을 원하고 있었다

## 📝 전후 요구사항 변화

### Before (clarify 전)
```
입금 매칭 자동화
- Bill.com ↔ Plaid 자동 매칭
- Slack/Notion 알림
- Dry-run → Production 전환
- Matching accuracy 95%+
```

### After (clarify 후)
```
입금 매칭 자동화 (명확해진 버전)

✅ 매칭 성공: 자동으로 Bill.com에 바로 반영 (승인 없이)
⚠️  Unmatched: Slack/Notion 즉시 알림 + 수동 처리 대기
❌ 95% 기준 제거: accuracy 숫자는 목표치가 아니라 모니터링 지표로 전환
🔧 오매칭 처리: 실패 기록 → 재매칭 시도 (단, Bill.com에 올라간 오류는 수동 수정)

발견된 갭:
- "잘못 매칭 → 자동 반영" + "잘못 매칭 → 재매칭 시도" 는 동시에 불가
  → 자동 반영 전 최소 검토 단계 필요할 수 있음 (추후 논의)
```

## 💭 오늘의 배움

**"당연하다고 생각한 것을 말로 꺼내는 순간, 그게 진짜 요구사항이 됩니다."**

이 문장이 Day 3 수업에서 가장 인상 깊었다.

나는 "95%+ 정확도"라고 적었지만, Claude에게 질문을 받기 전까지는:
- 95%가 안 됐을 때 어떻게 할지 생각한 적 없었다
- 잘못 매칭된 거래가 Bill.com에 올라가면 어떻게 되는지 생각한 적 없었다

**Claude의 질문이 내 머릿속의 암묵적 가정을 끌어냈다.**

clarify:vague의 핵심은 "오픈 질문 금지, 가설 선택지로 묻기"다.
"어떻게 할 건가요?" 대신 A/B/C로 물으니, 내가 무엇을 원하는지 빠르게 선택할 수 있었고,
동시에 "C) 생각 안 해봤다"를 고르는 순간 빠진 부분이 드러났다.
