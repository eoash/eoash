### 1️⃣ GitHub = AI 시대의 필수 인프라

**왜 비개발자도 GitHub을 써야 하나?**

AI를 잘 쓰려면 컨텍스트 품질이 중요한데, 컨텍스트는 결국 **문서와 파일**이다.
이 파일들을 체계적으로 관리하는 50년 역사의 도구가 Git/GitHub.

**핵심 용어 4가지:**

| 개념 | 의미 | 비유 |
|------|------|------|
| **Repository** | 프로젝트 폴더 | Slack 워크스페이스 |
| **Commit** | 특정 시점에 저장 | Save Point |
| **Branch** | 원본을 건드리지 않고 따로 작업 | 사본으로 작업하다가 나중에 합치기 |
| **PR (Pull Request)** | "이 변경사항 반영해도 될까?" 결재 | 결재 문서 |
| **Push** | 로컬 → 원격 저장소 반영 | 구글 드라이브에 업로드 |

**구글 드라이브와의 차이**:
- 구글 드라이브: 파일 공유 + 동시 편집
- GitHub: 파일 공유 + **변경 이력 추적** + **협업 리뷰(PR)**

### 2️⃣ Compound Engineering = 일할수록 효율이 올라가는 구조 만들기

**개념**: 매번 일한 뒤 시스템을 조금씩 개선해서, 시간이 지날수록 AI가 더 잘 돕는 상태를 만드는 것

**핵심 사이클**:
```
작업 → 완료 → Wrap → 개선 → 다음 작업 → 또 Wrap → 더 나은 개선
```

이걸 반복하면 CLAUDE.md가 정교해지고, 스킬이 쌓이고, 문서가 살아있는 상태가 유지된다.

**왜 중요한가?**
- 단일 AI 모델을 업그레이드하는 것보다
- 컨텍스트 품질을 높이는 것이 성능에 더 큰 영향을 줌
- 그 컨텍스트가 폴더 구조와 문서로 구현됨

**비유**: 인턴이 들어와서 아무 가이드 없이 일하면 문제가 생김. CLAUDE.md, 스킬, 폴더 구조가 잘 만들어질수록 AI 인턴이 점점 더 잘 일함.

### 3️⃣ Session Wrap Skill = 퇴근 전 책상 정리

**목적**: 세션 끝에 자동으로 "다음에 더 잘하기 위한 것들"을 뽑아내는 스킬

**작동 구조 (멀티 에이전트 2-Phase Pipeline)**:

```
세션 종료 → /wrap 실행

Phase 1: 4명의 전문가 에이전트 병렬 실행
┌─────────────────┬──────────────────┬─────────────────┬─────────────────┐
│ Document        │ Automation       │ Learning        │ Follow-up       │
│ Updater         │ Scout            │ Extractor       │ Suggester       │
├─────────────────┼──────────────────┼─────────────────┼─────────────────┤
│ 업데이트 필요한  │ 반복되는 작업    │ 이 세션에서     │ 다음에 할 일    │
│ 문서 찾기        │ 발견 → 스킬화?   │ 배운 것 정리    │ 제안            │
└─────────────────┴──────────────────┴─────────────────┴─────────────────┘

Phase 2: Duplicate Checker
→ Phase 1 결과 중 중복/모순 제거

최종: 사람이 리뷰 후 선택 → 문서 업데이트 + Commit
```

**중요**: 최종 실행은 사람이 승인. AI가 알아서 다 적용하지 않음.

**Session Wrap이 없으면**:
- 세션마다 개선점을 흘려보냄
- CLAUDE.md가 낡아짐
- 같은 실수 반복

**Session Wrap이 있으면**:
- 매 세션이 시스템을 개선하는 기회
- 스킬과 문서가 자동으로 최신 상태 유지
- 복리로 쌓임

## 🛠️ 실제로 어떻게 쓰나

**기본 흐름**:
1. Claude Code에서 작업
2. 작업 완료 후 `/wrap` 입력
3. 4개 에이전트가 동시 분석 (1-2분 소요, 토큰 많이 씀)
4. 제안 목록 확인 후 필요한 것만 선택
5. 자동 Commit

**추가 스킬 2가지**:

| 스킬 | 목적 | 사용 시점 |
|------|------|---------|
| `my-history-insight` | 과거 세션들 분석 | 주 1회, "지난주 뭘 했나?" |
| `my-session-analyzer` | 스킬 실행 기록을 SKILL.md와 비교해 PASS/FAIL 검증 | 스킬이 제대로 돌아가는지 확인 |

---

## ⚠️ 주의사항

- **토큰 비용**: Session Wrap은 서브에이전트 4개가 동시 실행 → 비용이 있음. 하지만 반복 작업 자동화로 장기적으로 상쇄됨
- **AI가 문서를 무분별하게 변경할 수 있음** → 반드시 사람이 최종 확인 후 Commit
- **폴더 구조가 중요**: CLAUDE.md를 어떤 폴더에서 켜느냐에 따라 AI가 읽는 컨텍스트가 달라짐. 폴더 설계가 컨텍스트 품질을 결정함
