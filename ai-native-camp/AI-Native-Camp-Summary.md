## 캠프 개요

AI Native Camp는 **비개발자도 Claude Code를 활용해 실제 업무 자동화 시스템을 직접 만드는 것**을 목표로 한 7일 집중 캠프. 단순 사용법 교육이 아닌, 각자의 실제 업무 문제를 가져와 AI 도구로 해결하는 실전 중심 과정.

**캠프의 4가지 참가자 유형**:
- **A형**: 반복 업무를 시스템으로 만들어 자동화하고 싶은 사람
- **B형**: 바이브 코딩으로 프로젝트를 만들고 싶은 사람
- **C형**: 배운 것을 조직 변화에 적용하고 싶은 리더
- **D형**: AI를 써야 하는데 어떻게 시작해야 할지 모르는 사람

## Day 1 — Claude Code 기초 & 환경 구축 (02/14, 금)

**참가자**: 45명+, 안서현 포함
**강사**: 장원준

### 핵심 교육 내용

#### 1. Claude Code CLI 설치 및 실행
- Desktop App보다 **CLI(터미널)가 더 빠르고 생산적**이라는 점 강조
- 기본 터미널 명령어 실습: `cd`, `ls`, `pwd`, `cat`, `open`
- 설치 장애 요인 실시간 해결: Git, Xcode, OS 버전 문제

#### 2. Git 통합
- `clone`, `commit`, `branch` 등 버전 관리 명령어 활용법
- 파일 시스템 관리 및 환경 설정 관련 문제 해결

#### 3. Skills (스킬)
- 스킬의 개념: **필요 시 최소한의 컨텍스트만 로드해 토큰 소비를 줄이는 구조**
- 반복 업무 자동화 레시피로 스킬 관리
- CLAUDE.md 업데이트와 연계하여 팀/개인 워크스페이스 관리

#### 4. Sub-agents (서브에이전트)
- 메인 에이전트의 컨텍스트를 분리해 별도 작업 환경 제공
- 병렬 작업 및 리서치 효율화
- 여러 에이전트 간 협업으로 복수 작업 단위 동시 관리

#### 5. Plugins (플러그인)
- User / Project / Local 3가지 스코프 개념
- 도구 배포 및 활성화 방법

### 다음 단계
- MCP 활용법 및 싱크 방법 추가 학습
- 각자 환경에 Claude Code, Git, CLI 설치 완료

## Day 2 — MCP 연동 & 컨텍스트 싱크 스킬 (02/16, 월)

**참가자**: 66명+
**발표**: 정구봉, 안철우

### 핵심 교육 내용

#### 1. 캠프 진행 방식 안내
- Slack 채널 이동 (`day-1` → `day-2`)
- 자기소개, 팟캐스트 코너, Q&A 운영 방식 설명

#### 2. Claude Code 폴더 구조 설계
- "엘바프 허브" 사례: 업무와 프로젝트 정보를 효율적으로 관리하는 폴더 구조
- 템플릿, 스킬 구성 및 리팩토링 방법론

#### 3. MCP (Model Context Protocol) 활용
- MCP 개념: Claude Code에 외부 도구를 연결하는 프로토콜
- **Slack 연동**: `npx-skills-add` 명령어로 스킬 설치 및 실행 시연
- **Enable Tool Search**: 토큰 과부하 문제 해결을 위한 컨텍스트 최적화 방법
- **공식 MCP vs 비공식 MCP**: 공식 MCP 도구 사용 권장 (성능 최적화 및 안정성)
- Slack 다중 워크스페이스 이슈: 한 세션 내 MCP 서버가 하나의 워크스페이스에만 바인딩되는 기술적 한계

#### 4. 마케팅 & CRM 자동화 사례 (최은영 Mia)
- 카카오톡 알림 + Slack 연동으로 CRM 메시지 발송 자동화
- 외주 효율화 방안 공유
- 이미지/텍스트 처리 방식 논의

#### 5. Gmail & Google Calendar 연동
- Google Cloud Console 설정 가이드
- Gmail, Calendar API 연동 실습 (다음 세션 예고)

#### 6. Git Clone으로 최신 스킬 설치
- 제공된 레포지토리 기반으로 최신 스킬 설치 안내

### 안서현의 실습 성과
- **my-context-sync 스킬 제작**: Slack, Notion, ClickUp, Gmail, Calendar 7개 소스를 병렬 수집해 하나의 마크다운 문서로 통합하는 스킬

**게스트 소개**:

**주요 토론 포인트**:

|------|--------|------|

### 핵심 교육 내용

#### 1. GitHub 기초 실습

#### 2. 컴파운드 엔지니어링 (Hyunjong Choi 패널)
핵심 개념: **스킬 만들기 → 운영 → 시스템화 → 지속 유지보수**

> 스킬 하나 만드는 것에서 끝나는 게 아니라, 그것을 운영하고 어떻게 시스템화해서 계속 유지보수하면서 갈지에 대한 사고방식

- 일회성 자동화 vs 지속 운영 가능한 시스템의 차이
- 안영학 대표: 인사평가 시스템 즉석 구현 사례

#### 3. Session Wrap 스킬 제작 (멀티에이전트 2-Phase Pipeline)

```
Phase 1 — 병렬 실행:
  ├── Document Updater    (문서 업데이트)
  ├── Automation Scout    (자동화 가능 항목 탐색)
  ├── Learning Extractor  (학습 내용 추출)
  └── Follow-up Suggestion (다음 단계 제안)

Phase 2 — 순차 실행:
  └── Duplicate Checker   (중복 항목 제거)
```

#### 4. 세션 분석 스킬들

**my-history-insight 스킬**:
- 과거 Claude Code 세션 기록을 3개 에이전트가 병렬 분석
- 작업 패턴, 반복 실수, 개선점 인사이트 추출

**my-session-analyzer 스킬**:
- SKILL.md 설계도 vs 실제 실행 기록 비교
- PASS/FAIL 검증 리포트 자동 생성

### 안서현의 실습 성과
- **my-session-wrap** 스킬 제작 (멀티에이전트 2-Phase Pipeline 직접 설계)
- **my-history-insight** 스킬 제작 (세션 히스토리 3개 에이전트 병렬 분석)
- **my-session-analyzer** 스킬 제작 (SKILL.md vs 실행 기록 검증)

- Notion을 지식 베이스로 Claude Code와 연동하는 방법
- 데이터베이스 설계 원칙

#### 2. AI 트렌드 소화법
- 빠르게 쏟아지는 AI 정보를 효율적으로 흡수하는 방법
- 콘텐츠 유형별 처리 전략 (트윗, 유튜브, 아티클)

```
X/Twitter URL 입력
  → 트윗 원문 가져오기
  → 요약 + 핵심 인사이트 + 한국어 번역 출력

```

```
콘텐츠 입력
  → 퀴즈 먼저 제시 (사전 지식 확인)
  → 정답/오답 확인
  → 틀린 부분 집중 학습

```

## Day 6 — 오프라인 세션: 컨설팅 패널 & GitHub 심화 (02/20, 금)

**장소**: 강남역 인근 삼성화재 빌딩 (오프라인)
**패널**: Gray Shin (전 BCG, 현 핀테크), 김태현 Taehyun Kim (BCG 3년차)
**사회**: 김성중

### 핵심 교육 내용

#### 1. BCG 컨설턴트 패널: AI가 컨설팅 업계에 미치는 영향

**컨설팅의 3개 레이어 구조**:

| 레이어 | 역할 | AI의 영향 |
|--------|------|-----------|
| 1년~3년차 (워킹 레벨) | 데이터 수집·분석, 컨텐츠 깎기 | 대부분 AI로 대체 가능 |
| PM | 스토리라인 구성, 설득 전략 수립 | 부분 AI 보조 |
| 파트너 | C레벨 설득, 의사결정 유도 | AI가 대체 불가 |

**주요 변화**:
- **RA(Research Assistant) 역할 소멸**: 단순 리서치·분석은 AI가 대체 → RA를 팀당 1명으로 축소, 조기 퇴근
- **스태핑 구조 변화**: 1+4 → **1+1 또는 1+2** (인원 감소, 프로젝트 수 증가)
- **Value-Based Pricing 등장**: 시간빌링 → **임팩트 기반 가격 책정** (컨설팅 ROI 직접 산정)
- **모든 레벨의 역할 상승**: AI로 인해 1~2년차도 스토리라인·클라이언트 설득 고민 요구
- **채용 변화**: 케이스 인터뷰 유지하되, **AI 활용 문제 해결 능력 평가** 추가 (맥킨지 사례)

**컨설팅이 살아남는 영역**:
> "팔란티어는 프로덕트는 좋은데 C레벨 설득이 안된다. 결국 변화관리, 전사 이니셔티브 수립은 트래디셔널 컨설팅이 여전히 강하다."
- C레벨 설득 및 변화관리 (Change Management)
- AI 도입 후 채용·인사·CDP 전면 재설계 (전략 컨설팅 역할)
- 기존 레거시 조직이 AI 도입 시 총체적 솔루션 제공

**전략적 사고 = 개발자 사고방식**:
> "컨설팅의 MECE는 개발자의 Divide and Conquer와 본질적으로 같다"
- Gray Shin (전 쿠팡페이 엔지니어 → BCG → 핀테크)의 핵심 통찰
- 문제를 쪼개고 구조화하는 방식은 동일

#### 2. GitHub 협업 심화 실습

## 핵심 개념 요약

### Claude Code 핵심 컴포넌트
| 컴포넌트 | 역할 |
|---------|------|
| CLAUDE.md | 프로젝트/세션 컨텍스트 파일 |
| Skill | 최소 컨텍스트로 로드되는 재사용 가능 레시피 |
| Sub-agent | 독립 컨텍스트로 병렬 작업 수행하는 서브 에이전트 |
| MCP | 외부 도구(Slack, Notion, GitHub 등) 연결 프로토콜 |
| Plugin | User/Project/Local 스코프의 확장 도구 |
| Hook | 도구 호출에 반응하는 자동화 트리거 |

### 멀티에이전트 패턴
- **병렬 실행**: 독립적인 작업을 동시에 처리해 속도 극대화
- **순차 검증**: 병렬 결과를 단일 에이전트가 통합·검증
- **오케스트레이터-워커**: 조율 에이전트 + 실행 에이전트 분리
- **컨텍스트 손실 방지**: 파일 저장 + 병합 패턴

### Clarify 프레임워크 (3가지)
| 스킬 | 사용 상황 |
|------|---------|
| clarify:vague | 요구사항이 모호할 때 → 구체적 스펙으로 정련 |
| clarify:unknown | 전략의 블라인드 스팟 탐색 (Known/Unknown 4분면) |
| clarify:metamedium | 내용 vs 형식 관점 전환 (Alan Kay 메타미디엄) |

## 캠프 전체 인사이트

1. **비개발자도 시스템 빌더가 될 수 있다**: 비개발자 45명이 7일 만에 자동화 시스템 직접 구축
2. **70% 완성도로 먼저 작동**: 완벽한 계획보다 불완전해도 작동하는 것이 중요
3. **사람이 AI에게 질문하는 게 아니라, AI가 사람에게 질문해야 한다**: Clarify가 핵심
4. **컴파운드 엔지니어링**: 스킬은 만들고 끝이 아니라 운영·시스템화·유지보수까지
5. **집단 지성의 가치**: 혼자 Claude Code와 하는 것보다, 45명이 함께 하는 것이 더 빠른 성장
6. **프로젝트 에이전트(자동화) vs 어드바이저(판단) 구분**: 자동화할 것과 판단이 필요한 것을 분리
