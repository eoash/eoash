# Dev Team Agent Skill 설계서

**작성일**: 2026-02-24
**목적**: 기획 → 아키텍처 → 개발 → 리뷰 4단계 에이전트 팀으로 기능을 자동 개발하는 스킬

---

## 개요

사용자가 기능 설명을 입력하면, 4명의 에이전트가 순차적으로 실행되며 PRD → 설계 → 코드 → 리뷰까지 자동 완성한다.

## 팀 구성

| 에이전트 | 역할 | 입력 | 출력 |
|---------|------|------|------|
| PM (기획자) | 요구사항 → PRD 작성 | 사용자 기능 설명 | `docs/PRD.md` |
| 아키텍트 | PRD → 기술 설계 | `PRD.md` | `docs/ARCH.md` |
| 개발자 | ARCH → 코드 구현 | `PRD.md` + `ARCH.md` | 코드 파일들 |
| 리뷰어 | 코드 검토 + 개선 | 코드 + `PRD.md` | `docs/REVIEW.md` + 수정 코드 |

## 실행 흐름

```
사용자 입력 → PM → [PRD 검증] → 아키텍트 → [ARCH 검증] → 개발자 → [코드 검증] → 리뷰어 → 최종 보고
```

## 검증 규칙

| 단계 | 검증 기준 | 실패 시 |
|------|---------|--------|
| PRD 검증 | 배경/기능 요구사항/성공 기준 섹션 존재 여부 | PM 재실행 (최대 1회) |
| ARCH 검증 | 컴포넌트/파일 구조/구현 계획 섹션 존재 여부 | 아키텍트 재실행 (최대 1회) |
| 코드 검증 | ARCH에서 명시한 파일들 실제 생성 여부 | 개발자 재실행 (최대 1회) |

## 결정 사항

- 접근법: B (파이프라인 + 단계별 검증)
- 팀 규모: 고정 4명
- 승인 방식: 완전 자동 실행
- 결과물: 코드 + 문서 둘 다
