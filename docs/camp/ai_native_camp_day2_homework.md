# AI Native Camp Day 2 과제 - 안서현

## 질문 1: 오늘 다룬 도구 중 첫 번째로 연결한 도구 + 이유

### 🎯 첫 번째 선택: **Slack**

**이유:**
- EO Studio는 한국/미국/베트남 3개 지역에 분산되어 있어서 Slack이 가장 중요한 소통 채널입니다
- 15개 채널(공지, 운영, 비즈니스, 글로벌 전략, AI Lab 등)에서 매일 수십 개의 의사결정이 일어나는데, 놓치는 정보가 많았습니다
- 특히 `2026-global-heads` 같은 프라이빗 채널의 전략적 논의를 체계적으로 추적할 방법이 필요했습니다

### 📋 기존 Workflow와 문제점

**Before (수동 방식):**
1. 아침에 Slack 열어서 15개 채널 일일이 스크롤
2. 중요한 메시지 복사해서 Notion에 수동으로 정리
3. 멘션된 항목 놓치는 경우 발생
4. 시간 소요: 매일 30분+

**문제점:**
- 시차 때문에 미국팀 메시지를 아침에 한꺼번에 확인
- "확정", "결정" 같은 중요 키워드가 묻히는 경우
- 여러 채널에 흩어진 맥락을 연결하기 어려움

**After (Claude Code + MCP):**
1. `/sync` 명령 한 번으로 15개 채널 자동 수집
2. AI가 자동으로 중요도별 분류 (긴급/공지/의사결정/멘션)
3. 하나의 마크다운 문서로 통합
4. 시간 소요: 5분 이내

**실제 효과:**
- 오늘 아침 싱크 결과에서 "야놀자 5,500만원 선금 결제 기한 2/27"이라는 놓칠 뻔한 중요 정보를 즉시 파악
- 글로벌 전략 채널의 "콘텐츠 투자상품 리스크 관리" 논의를 전체 맥락과 함께 확인

## 질문 2: 오늘 언급되지 않은 도구 중 연결하고 싶은 도구

### 💡 추가 연결 도구: **ClickUp**

**이유:**
- EO Studio에서 프로젝트 관리와 개인 태스크 모두 ClickUp으로 관리합니다
- 제가 Finance Lead이다 보니 마감이 있는 재무 태스크(정산, 청구서, 급여)가 많은데, 마감 임박 항목을 놓치면 큰 문제가 됩니다
- 하지만 ClickUp은 강의에서 다루지 않아서 직접 연결해야 했습니다

### 🔧 실제 연결 과정

**Step 1: ClickUp API 연구**
- ClickUp REST API 문서 확인
- Personal API Token 발급

**Step 2: Python 스크립트 작성**
- `clickup_fetch.py` 직접 구현
- 나에게 할당된 태스크, 진행 중인 태스크, 마감 임박 태스크 수집

**Step 3: Context Sync 스킬에 6번째 소스로 추가**
- `my-context-sync/SKILL.md` 업데이트
- 실행 흐름에 ClickUp 병렬 수집 추가

**Step 4: 기능 확장 (오늘 추가 구현)**
- "due date이 없는 태스크"도 수집 (백로그 관리용)
- "오늘 내가 생성한 태스크" 별도 표시 (급하게 추가한 항목 추적)

### 📊 연결 전/후 차이

**Before:**
- ClickUp 앱 열어서 직접 필터링
- "마감 지난 태스크"를 일일이 찾아야 함
- Finance 폴더와 Personal List를 따로 확인

**After:**
```
✅ ClickUp 태스크 수집 완료!

통계:
- 총 12개 (진행 중 0개, 마감 임박 10개)
- Due date 없음: 2개
- 오늘 생성: 1개 (미국총무카드 업데이트)

⚠️ 긴급: 마감 지난 Finance 태스크 3건
- 한국 계정과목 수정요청 (2/12 마감)
- 12월 미국 정산 자료 생성 (2/11 마감)
- 고위드 1월 카드사용내역 승인처리 (2/11 마감)
```

**핵심 개선:**
- 아침에 `/sync` 한 번으로 Slack + Calendar + Notion + ClickUp을 한눈에 파악
- "오늘 생성한 태스크" 기능으로 급하게 추가한 일감을 놓치지 않음
- 15개 Slack 채널 + 12개 ClickUp 태스크 + 8개 Calendar 일정을 5분 만에 정리

## 💭 오늘의 배움

가장 중요한 깨달음은 **"도구는 연결되어야 의미가 있다"**는 것입니다.

Slack, Notion, ClickUp, Calendar를 각각 열어서 확인할 때는 맥락이 흩어졌습니다.
하지만 6개 도구를 하나의 Context Sync 문서로 통합하니, 비로소 "전체 그림"이 보였습니다.

예를 들어 오늘:
- **Slack**: 야놀자 5,500만원 선금 청구서 발행
- **Calendar**: 2/20 운영팀 위클리 일정
- **ClickUp**: 12월 미국 정산 자료 생성 마감 지남

이 3개가 따로 있을 때는 그냥 "할 일 목록"이었는데,
통합 문서에서 보니 "운영팀 위클리 전에 정산 자료를 먼저 끝내야 한다"는 우선순위가 명확해졌습니다.

**AI Native Workflow = 도구를 연결하고, 맥락을 통합하고, 액션을 자동화하는 것**
