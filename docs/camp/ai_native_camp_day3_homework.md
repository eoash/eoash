# AI Native Camp Day 3 과제 - 안서현

## 질문 1: 오늘 다룬 도구 중 첫 번째로 사용한 도구 + 이유

### 🎯 첫 번째 선택: **clarify:unknown** (Known/Unknown 4분면 분석)

**이유:**
- EO Studio에서 AI Native 전략을 짜고 있는데, "뭘 놓치고 있는지 모른다"는 느낌이 항상 있었습니다
- 지금 하는 일이 정말 올바른 방향인지, 보이지 않는 리스크가 있는지 체계적으로 점검하고 싶었습니다
- clarify:unknown의 4분면 구조가 딱 맞았습니다: 내가 아는 것, 내가 모르는 것, 내가 모른다는 걸 모르는 것

### 📋 적용한 전략: EO Studio AR 자동화 시스템

**분석 대상 전략:**
- 매일 2시간 걸리는 입금 매칭 수작업을 Bill.com + Plaid 자동화로 대체
- Phase 1 MVP 이후 Phase 2 고도화 계획 중

**4분면 분석 결과:**

| 분면 | 내용 |
|------|------|
| **Known Knowns (확실한 것)** | Bill.com API 연동 완료, Plaid로 입금 내역 수집 가능, 3단계 매칭(Exact→Number→Fuzzy) |
| **Known Unknowns (모르는 것을 알고 있음)** | 매칭 정확도가 실제로 95%인지, 엣지케이스 처리, Hanmi Bank 연동 |
| **Unknown Knowns (모른다고 생각했는데 이미 아는 것)** | Slack 알림 시스템 이미 있음, pytest 테스트 커버리지 이미 확보 → 이걸 AR 시스템에 바로 적용 가능 |
| **Unknown Unknowns (모른다는 것조차 몰랐던 것)** | 은행 거래 내역에 "메모란"이 없는 경우, 한국 원화 → 달러 환전 시 타이밍 문제 |

**가장 큰 발견:**
> "Unknown Knowns"에서 예상치 못한 자산을 발견했습니다.
> 이미 만들어둔 Slack 통보 시스템과 테스트 커버리지가 AR 자동화에 즉시 재활용 가능했는데,
> 새로 만들어야 한다고 생각하고 있었습니다.

### 📊 Before / After

**Before (수동 방식):**
- AR 추적: Excel 파일에서 직접 입금 대조
- 매일 아침 2시간 수작업
- "이게 맞나?" 불안함

**After (clarify:unknown 적용 후):**
```
Known Unknowns 3개 식별 → 실험 설계
- 매칭 정확도 95% 가정 → 테스트 데이터로 검증 필요
- Hanmi Bank 연동 → Phase 2로 명시적 이동
- 환율 타이밍 → 일단 달러 기준으로 고정 후 추후 개선

Stop Doing 1개:
- 수동 Plaid 데이터 다운로드 → 완전 자동화로 대체
```

## 질문 2: 오늘 언급되지 않은 시나리오 중 적용하고 싶은 것

### 💡 추가 적용: **clarify:metamedium** — 재무 보고서의 "형식" 자체를 바꾸기

**이유:**
- 매월 3개국(한국/미국/베트남) 재무 데이터를 정리해서 CEO에게 보고합니다
- 지금까지는 "보고서 내용(Content)"을 어떻게 잘 쓸까만 고민했습니다
- 오늘 metamedium 개념을 배우고 나서 깨달았습니다: 내가 최적화해야 하는 건 내용이 아니라 **형식(Form)** 자체입니다

### 🔧 Content vs Form 분석

**현재 방식 (Content 최적화):**
- Excel로 3개국 데이터 수동 집계
- PPT/Notion 보고서 직접 작성
- 매달 반복, 매달 새로 만들어야 함

**Form 전환 후 (자동 생성 파이프라인):**
- Bill.com + 한국 회계 + 베트남 데이터 → 자동 수집
- Claude로 자동 요약 + 시각화
- CEO에게 자동 발송

### 📊 연결 전/후 차이

**Before:**
```
3개국 재무 데이터를 수동으로 모아서
→ 엑셀에서 정리하고
→ PPT로 만들어서
→ 매달 보고
(소요 시간: 1일/월)
```

**After:**
```
Claude Code + 3개국 API
→ 자동 집계 + 요약
→ Slack/Notion으로 자동 전송
(소요 시간: 5분 검토/월)

🎯 Form Opportunity:
- 내용: 보고서 잘 쓰기 → 이미 잘 하고 있음
- 형식: 수동 보고서 → 자동 생성 파이프라인
- 최소 실험: Bill.com 데이터만으로 먼저 자동화 테스트
```

**핵심 깨달음:**
> "내용을 더 잘 쓰려고 했던 게 잘못된 방향이었다.
> 형식 자체를 자동화 파이프라인으로 바꾸면
> 내용은 AI가 알아서 잘 써준다."

## 💡 bonus: clarify:vague 실제 적용 사례

Day 3에서 clarify:vague로 AR 자동화의 모호한 요구사항을 정리했습니다:

**Before (모호한 요구사항):**
> "입금 매칭 자동화해줘"

**clarify:vague 적용 후 (명확한 스펙):**
```markdown
## Requirement Clarification Summary

### Before
"입금 매칭 자동화해줘"

### After
**Goal**: Bill.com 미수금과 Plaid 입금 내역을 자동 매칭해서 매일 오전 9시 Slack으로 결과 전송
**Scope**: 미국 법인 USD 거래만 (한국/베트남 제외)
**Constraints**: 매칭 정확도 95%+, Dry-run 모드 필수, 수동 오버라이드 가능해야 함
**Success Criteria**: 매일 아침 2시간 → 10분 검토로 단축

**Decisions Made**:
| 모호한 부분 | 결정 사항 |
|-------------|----------|
| 어떤 법인? | 미국 법인 먼저 (Phase 1) |
| 자동 업데이트? | Dry-run 먼저, 검토 후 수동 승인 |
| 알림 채널? | Slack DM (서현에게만) |
```

## 💭 오늘의 배움

가장 중요한 깨달음은 **"질문의 질이 답의 질을 결정한다"**는 것입니다.

clarify 도구 3가지가 각각 다른 수준에서 질문을 정제합니다:

| 도구 | 질문 수준 | 내가 쓰는 시점 |
|------|----------|--------------|
| **vague** | 요구사항이 모호할 때 | "이거 만들어줘" 같은 말이 나올 때 |
| **unknown** | 전략/방향이 불확실할 때 | "맞는 방향인지 모르겠어" 할 때 |
| **metamedium** | 같은 방식이 한계에 왔을 때 | "더 잘해도 나아지는 게 없어" 할 때 |

특히 **metamedium**의 Alan Kay 명언이 인상 깊었습니다:
> "A change of perspective is worth 80 IQ points."

재무 보고서를 더 잘 쓰려는 노력보다, 보고서 자체를 자동 생성되는 형식으로 바꾸는 것이
80배의 레버리지를 만든다는 뜻이었습니다.

**AI Native Workflow = 내용을 최적화하기 전에, 형식 자체를 바꿀 수 있는지 먼저 물어보기**
