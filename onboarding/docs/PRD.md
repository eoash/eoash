# PRD: 신규 입사자 온보딩 시스템

**작성자**: 안서현 (Finance & Operations Lead, EO Studio)
**작성일**: 2026-02-24
**버전**: v1.0
**상태**: Draft

---

## 배경

신규 입사자가 조직에 합류했을 때, 회사의 맥락과 문화를 파악하는 데 평균 2~4주가 소요된다. 그 기간 동안 기존 팀원들은 반복적인 온보딩 설명에 시간을 쏟게 되고, 신규 입사자는 흩어진 정보를 스스로 찾아야 하는 부담을 안는다. Slack 채널과 Notion 문서에 이미 회사의 중요한 맥락이 축적되어 있음에도 이를 체계적으로 활용하지 못하고 있으므로, 이 데이터를 자동으로 수집·분석하여 신규 입사자가 첫 주 안에 핵심 맥락을 파악할 수 있는 시스템이 필요하다.

---

## 기능 요구사항

### F1. Slack 데이터 수집
- Slack API를 통해 지정된 채널(예: #general, #announcements, #team-*)의 최근 90일 메시지를 수집한다.
- 수집 대상: 채널 메시지, 핀된(pinned) 메시지, 파일 링크 포함 메시지.
- 수집 결과는 채널별 JSON 파일로 로컬에 저장한다.

### F2. Notion 데이터 수집
- Notion API를 통해 지정된 워크스페이스 내 페이지(공지사항, 팀 위키, 정책 문서 등)를 수집한다.
- 수집 대상: 페이지 제목, 본문 텍스트, 마지막 수정일.
- 수집 결과는 페이지별 JSON 파일로 로컬에 저장한다.

### F3. 핵심 맥락 자동 분석 및 요약
- 수집된 Slack + Notion 데이터를 분석하여, 신규 입사자에게 필요한 핵심 맥락을 자동으로 추출한다.
- 출력 항목: 회사 문화 키워드 Top 10, 최근 주요 공지사항 5건, 자주 언급된 프로젝트/팀 이름, 반드시 읽어야 할 Notion 문서 3~5개.
- 분석 결과는 Markdown 형식의 요약 리포트로 저장한다.

### F4. 신규 입사자용 온보딩 가이드 생성
- 분석 결과를 바탕으로 신규 입사자 전용 온보딩 가이드 문서(`ONBOARDING_GUIDE.md`)를 자동 생성한다.
- 가이드 구성: 회사 소개 요약 / 팀 구조 / 필독 문서 링크 / 알아야 할 Slack 채널 목록 / 첫 2주 체크리스트.
- 가이드는 Notion에 자동 업로드되거나 로컬 파일로 제공한다.

### F5. 갱신 자동화 (스케줄 실행)
- Python 스크립트를 cron 또는 GitHub Actions로 주 1회 자동 실행하여 데이터를 최신 상태로 갱신한다.
- 갱신 완료 시 지정된 Slack 채널(예: #hr-ops)에 완료 알림을 발송한다.

## 비기능 요구사항

### 성능
- 데이터 수집 및 분석 전체 프로세스는 10분 이내에 완료되어야 한다.
- Slack API Rate Limit(Tier 3: 50 req/min)을 준수하며 요청 간 딜레이를 적용한다.

### 보안
- Slack Bot Token, Notion API Key 등 모든 인증 정보는 `.env` 파일에 저장하며, `.gitignore`에 포함하여 코드 저장소에 커밋하지 않는다.
- 수집된 데이터는 로컬 또는 지정된 내부 경로에만 저장하며, 외부 서버로 전송하지 않는다.
- 개인정보(이름, 이메일 등)가 포함된 메시지는 수집 단계에서 마스킹 처리한다.

### 호환성
- Python 3.10 이상에서 동작해야 한다.
- macOS, Windows, Linux 모두에서 실행 가능해야 한다 (크로스 플랫폼).
- 의존 패키지는 `requirements.txt`에 명시하고, `pip install -r requirements.txt`로 설치 가능해야 한다.

### 유지보수성
- 수집 대상 채널 및 Notion 페이지 ID는 설정 파일(`config.yaml`)에서 관리하여 코드 수정 없이 변경 가능해야 한다.
- 각 스크립트는 단일 책임 원칙을 따르며 모듈화한다 (수집 / 분석 / 리포트 생성 분리).

## 성공 기준

| 지표 | 목표 |
|------|------|
| 온보딩 가이드 자동 생성 성공률 | 스크립트 실행 시 오류 없이 완료 100% |
| 신규 입사자 맥락 파악 기간 | 기존 2~4주 → 1주 이내 (자기 보고 기준) |
| 기존 팀원의 온보딩 설명 시간 | 1인당 주당 2시간 → 30분 이하 |
| 데이터 최신성 | 가이드가 항상 7일 이내 데이터 반영 |
| 필독 문서 클릭률 | 가이드 내 추천 문서 3건 이상 신규 입사자 열람 확인 |

## 범위 외 (Out of Scope)

- **실시간 Q&A 챗봇**: 신규 입사자가 질문을 입력하면 답변하는 인터랙티브 기능은 이번 범위에 포함하지 않는다.
- **Google Drive / Gmail 연동**: 이번 버전은 Slack과 Notion만 대상으로 하며, 다른 툴 연동은 추후 검토한다.
- **권한 관리 UI**: 수집 대상 채널이나 문서를 웹 UI에서 설정하는 기능은 제외한다. 설정 파일(`config.yaml`) 직접 수정으로 대체한다.
- **다국어 지원**: 한국어 기준으로 구현하며, 영어 등 다국어 처리는 이번 범위에 포함하지 않는다.
- **HR 시스템 연동**: 입사자 정보를 HR 시스템(예: 에어테이블)에서 자동으로 가져오는 기능은 제외한다.
- **자동 발송**: 생성된 온보딩 가이드를 신규 입사자에게 자동 이메일/Slack DM으로 발송하는 기능은 제외한다. 담당자가 수동으로 공유한다.
